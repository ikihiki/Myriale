# user-management-user-stories.md
## ユーザー管理に関するユーザーストーリー

本ドキュメントは、TextRpg のユーザー管理（認証・認可・プロフィール・退会・管理画面・OAuth）を
ユーザーストーリーとして定義する。

前提用語  
- User: サービス利用者  
- Admin: 管理者権限を持つ運用者  
- Profile: ユーザーの表示名・アイコン・設定などのプロフィール情報  
- OAuth: 外部IDプロバイダ（Google等）による認証  
- Session: （認証の）ログインセッション  

---

## US-UM01: 新規登録したい（メール/パスワード）

As a 利用者  
I want メールアドレスとパスワードで新規登録したい  
So that アカウントを作成してサービスを利用できる  

背景・意図  
- 最小の導線で登録できる必要がある  

ユーザー行動  
- メールアドレスとパスワードを入力して登録する  

期待される結果  
- アカウントが作成される  
- 初回ログイン状態になる（またはログイン画面に遷移する）  

制約  
- メールアドレスは一意  
- パスワードは要件（長さ/複雑さ）を満たす  

---

## US-UM02: 新規登録時にメール確認をしたい（任意）

As a 運営者  
I want メール確認を行いたい  
So that なりすましや不正登録を減らせる  

ユーザー行動  
- 確認メールのリンクを開く  

期待される結果  
- メール確認済みフラグが立つ  
- 未確認ユーザーの利用制限が適用される（任意）  

---

## US-UM03: ログインしたい（メール/パスワード）

As a 利用者  
I want メールアドレスとパスワードでログインしたい  
So that 自分のデータにアクセスできる  

ユーザー行動  
- 資格情報を入力してログインする  

期待される結果  
- 認証が成功し、セッションが開始される  
- 自分のシナリオ/セッション/ノートにアクセスできる  

例外  
- 認証失敗時は分かりやすいエラーメッセージが表示される  

---

## US-UM04: ログアウトしたい

As a 利用者  
I want ログアウトしたい  
So that 共有端末でも安全に利用できる  

ユーザー行動  
- ログアウトを選択する  

期待される結果  
- 認証セッションが無効化される  
- ログイン画面に戻る  

---

## US-UM05: パスワードをリセットしたい

As a 利用者  
I want パスワードを忘れたときにリセットしたい  
So that アカウントを失わずに復帰できる  

ユーザー行動  
- メールアドレスを入力してリセットを開始する  
- 送付されたリンクから新パスワードを設定する  

期待される結果  
- 新パスワードでログインできる  

制約  
- リセットリンクは期限付き  
- リセット要求がアカウントの存在有無を過度に漏らさない（任意）  

---

## US-UM06: OAuthで登録/ログインしたい

As a 利用者  
I want OAuth（例: Google）で登録/ログインしたい  
So that パスワード管理なしで利用できる  

ユーザー行動  
- OAuthプロバイダを選択し認可する  

期待される結果  
- 外部IDと紐づくアカウントが作成またはログインされる  
- 次回以降も同じプロバイダでログインできる  

例外  
- 既存メール/パスワードアカウントとメールが一致する場合、統合導線が必要（US-UM07）  

---

## US-UM07: OAuthアカウントと既存アカウントを統合したい（任意だが実務上重要）

As a 利用者  
I want OAuthログインを既存アカウントに紐づけたい  
So that データが分裂しない  

背景・意図  
- 同一メールで別アカウントができると運用事故になりやすい  

期待される結果  
- 本人確認を経て、同一Userにログイン手段が追加される  
- 既存のシナリオ/セッション/ノートは保持される  

---

## US-UM08: プロフィールを閲覧したい

As a 利用者  
I want 自分のプロフィールを見たい  
So that 登録情報や設定を確認できる  

プロフィール項目例  
- 表示名  
- アイコン  
- 自己紹介（任意）  
- 言語/表示設定  
- 通知設定（ノート更新通知など）  

期待される結果  
- 現在のプロフィール値が表示される  

---

## US-UM09: プロフィールを編集したい

As a 利用者  
I want プロフィールを編集したい  
So that 表示名や設定を自分好みにできる  

ユーザー行動  
- 表示名・アイコン等を更新して保存する  

期待される結果  
- 保存後、UI上の表示に反映される  
- バリデーションエラーは分かりやすく表示される  

---

## US-UM10: アカウントのセキュリティ設定を管理したい（任意）

As a 利用者  
I want セキュリティ設定を管理したい  
So that 不正利用リスクを下げられる  

例  
- ログイン履歴の閲覧  
- 他端末ログアウト  
- 2FA（将来拡張）  

期待される結果  
- 設定変更が反映される  

---

## US-UM11: アカウントを削除（退会）したい

As a 利用者  
I want アカウントを削除したい  
So that サービス利用を終了できる  

背景・意図  
- 個人データ削除の要求に応える必要がある  

ユーザー行動  
- 退会画面で注意事項を確認する  
- 本人確認（再認証）を行う  
- 削除を確定する  

期待される結果  
- アカウントが削除される（または一定期間の論理削除後に削除）  
- ログインできなくなる  

データ要件（方針選択が必要）  
- シナリオ/セッション/ノートを完全削除する  
- または匿名化して保持する（公開物がある場合）  

---

## US-UM12: 退会前にデータをエクスポートしたい（任意だが推奨）

As a 利用者  
I want 退会前に自分のデータをエクスポートしたい  
So that 手元に記録を残せる  

期待される結果  
- シナリオ/セッションログ/ノートがエクスポートできる  
- 形式はMarkdown/JSON等  

---

## US-UM13: 管理者としてユーザー一覧を管理したい（管理画面）

As a Admin  
I want ユーザー一覧を閲覧・検索したい  
So that 運用・サポートができる  

期待される結果  
- ユーザーの基本情報（ID、登録日、最終ログイン等）が見られる  
- 条件検索ができる  

---

## US-UM14: 管理者として不正ユーザーを制限したい（管理画面）

As a Admin  
I want ユーザーを停止/制限したい  
So that 規約違反や不正利用に対応できる  

ユーザー行動（Admin）  
- 対象ユーザーを選び、状態を変更する（停止/復帰）  

期待される結果  
- 停止中ユーザーはログイン不可、または機能制限される  
- 変更履歴が監査ログに残る  

---

## US-UM15: 管理者としてサポート対応のためユーザー情報を参照したい（管理画面）

As a Admin  
I want ユーザーの状態を参照したい  
So that 問い合わせ対応ができる  

期待される結果  
- 課金があるなら課金状態、なければ利用状況（セッション数等）を参照できる  
- 個人情報のアクセスは権限制御される  

---

## US-UM16: 監査ログを残したい（管理・セキュリティ）

As a 運営者  
I want 認証・権限・削除などの重要操作を監査ログに残したい  
So that インシデント対応と追跡ができる  

対象例  
- ログイン/失敗  
- 退会  
- Adminによる停止/復帰  
- OAuth連携/解除  

期待される結果  
- 監査ログが検索できる  
- 改ざん耐性を考慮できる（将来拡張）  

---

## 総括

- ユーザー管理は  
  - 登録/ログイン（メール・OAuth）  
  - プロフィール（読み書き）  
  - 退会（削除方針を含む）  
  - 管理画面（検索・停止・参照）  
  を中核とする  
- OAuthはアカウント統合が事故りやすいため、統合ストーリーを先に定義しておく
