# mode-transition-and-exception-user-stories.md
## モード遷移・例外系ユーザーストーリー

本ドキュメントは、セッション進行中に発生する
「AI対話モード」と「プログラム主導モード（バトル・判定・強制イベント等）」の
切り替え、およびその途中で発生しうる例外・中断・復帰を
安全かつ分かりやすく扱うためのユーザーストーリーを定義する。

対象となるモード  
- AI対話モード（自由入力）  
- プログラム主導モード（選択UI・ダイス・強制進行）  

---

## US-M01: システムが進行モードを明示的に切り替えたい

As a システム  
I want セッションの進行モードを明示的に切り替えたい  
So that 処理方法とUIの責務が混ざらない  

背景・意図  
- バトルや判定中に自由入力があると処理不能になる  
- モード切替は暗黙ではなく明示すべき  

トリガー例  
- バトル開始  
- ダイス判定開始  
- 強制イベント開始  

期待される結果  
- 現在のモードが Session State に保存される  
- UIがモードに応じて切り替わる  
  - AI対話モード：自由入力有効  
  - プログラム主導モード：自由入力無効  

---

## US-M02: プレイヤーが現在の進行モードを理解できるようにしたい

As a プレイヤー  
I want 今どの進行モードか分かるようにしたい  
So that 操作に迷わない  

背景・意図  
- 入力できない理由が分からないと不満になる  

期待される結果  
- UI上に現在のモードが明示される  
  - 「対話中」  
  - 「バトル中」  
  - 「判定中」  
  - 「イベント進行中」  
- 自由入力不可の場合、その理由が分かる  

---

## US-M03: プログラム主導モードが正常終了したらAI対話に戻りたい

As a プレイヤー  
I want 強制進行が終わったら自然に対話へ戻りたい  
So that 物語を自由に続けられる  

背景・意図  
- モード復帰が曖昧だと操作不能になる  

期待される結果  
- プログラム主導シーン完了時  
  - モードが AI対話 に戻る  
  - 自由入力が再度有効になる  
- 復帰時の状況が Narrative として提示される  

---

## US-M04: プログラム主導モード中にエラーが起きても安全に復帰したい

As a プレイヤー  
I want 途中でエラーが起きても進行不能にならないようにしたい  
So that セッションが壊れない  

背景・意図  
- プログラム主導処理は失敗点が多い  
- 中途半端な状態が残ると復旧できない  

期待される結果  
- エラー発生時  
  - どこまで確定したかが明示される  
  - 再試行または安全な地点に戻れる  
- セッション状態が破損しない  

制約  
- 未確定の処理結果は反映されない  

---

## US-M05: プログラム主導モード中に通信断・画面離脱が起きても再開したい

As a プレイヤー  
I want 途中で離脱しても同じ状態から再開したい  
So that やり直しにならない  

背景・意図  
- モバイルやWebでは中断は避けられない  

期待される結果  
- 再接続時  
  - 最後に確定した進行地点から再開される  
  - 未完了の処理は再提示される  
- モード状態が正しく復元される  

---

## US-M06: プログラム主導モード中は巻き戻し操作を制限したい

As a システム  
I want 強制進行中は巻き戻しを制限したい  
So that 状態不整合を防げる  

背景・意図  
- バトルや判定途中の巻き戻しは矛盾を生む  

期待される結果  
- プログラム主導モード中  
  - 巻き戻しUIは無効化される  
  - または「終了後に可能」と明示される  

補足  
- モード終了後は通常通り巻き戻し可能  

---

## US-M07: モード遷移の履歴をログとして残したい

As a 開発者  
I want モード遷移をログに残したい  
So that デバッグや不具合解析ができる  

背景・意図  
- 「なぜ入力できなかったか」は後から確認したい  

期待される結果  
- 各モード遷移に  
  - 遷移理由  
  - 開始/終了時刻  
  が記録される  

---

## US-M08: 強制進行中でも最低限の情報確認はできるようにしたい

As a プレイヤー  
I want 進行中でも状況を確認したい  
So that 置いていかれた感じにならない  

背景・意図  
- 完全ロックは不安を生む  

期待される結果  
- 強制進行中でも  
  - 現在の目的  
  - 処理中内容（例：ダイス判定中）  
  が確認できる  

---

## 総括

- モード遷移は「暗黙」ではなく「状態」として管理する  
- プログラム主導モードは  
  - 自由入力無効  
  - 巻き戻し制限  
  - 安全な復帰  
  を前提に設計する  
- 例外系（エラー・中断）は  
  「どこまで確定したか」を軸に復旧する  
- 本ストーリー群は  
  バトル・判定・強制イベントすべての共通基盤となる
