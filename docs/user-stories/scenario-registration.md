# scenario-registration.md
## シナリオ登録に関するユーザーストーリー

## US-01: 新しいシナリオを作成したい

As a プレイヤー / シナリオ作者  
I want 新しいシナリオを作成したい  
So that 自分専用、または他人に共有できるRPG体験の土台を作れる

背景・意図  
- シナリオはすべてのプレイ体験の起点である  
- 初期段階では最小限の入力で作成できる必要がある  
- 作成時点では内容が未完成であることが前提  

前提条件  
- ユーザーがログインしている  
- シナリオ作成権限を持っている  

ユーザー行動  
1. 「新しいシナリオを作成」を選択する  
2. シナリオタイトルを入力する  
3. 任意で概要（あらすじ）を入力する  
4. 保存を実行する  

期待される結果  
- シナリオが「Draft（下書き）」状態で保存される  
- システムが一意な ScenarioId を発行する  
- 以降の設定画面に遷移できる  

非機能・補足  
- タイトルのみ必須  
- 概要は空でもよい  
- 保存後すぐにAI補助機能を利用できる  

## US-02: シナリオのジャンルをタグで指定したい

As a シナリオ作者  
I want タイトルの近くでジャンルをタグとして指定したい
So that シナリオの分類と物語の方向性を簡潔に示せる

背景・意図
- AIはジャンル情報がないと出力が不安定になりやすい
- 世界観や雰囲気は基本情報へ集約し、入力先を分散させない

ユーザー行動
- シナリオタイトル直下のジャンルタグを入力する
- 世界観や雰囲気は基本情報へMarkdownで記述する

期待される結果
- ジャンルが ScenarioSetting として保存される
- ジャンルタグがタイトルと一体で確認できる
- 基本情報が Narrative生成・挿絵生成の共通文脈として参照される

補足
- ジャンルタグは未指定でもよい

## US-04: AIの裁量レベルを調整したい

As a シナリオ作者  
I want AIの自由度を制御したい  
So that 物語が暴走せず、かつ退屈にもならない

背景・意図  
- AIの創造性は制御しなければ破綻する  
- ユーザーの許容範囲は人によって違う  

ユーザー行動  
- AI裁量レベル（低〜高）を選択する  

期待される結果  
- Narrative生成時の制御パラメータとして使用される  

補足  
- UIで挙動差を明示  
- セッション開始後の変更は将来拡張  

## US-05: 初期キャラクター条件を設定したい

As a シナリオ作者  
I want 主人公の前提条件を決めたい  
So that 導入時の説明を簡略化できる

背景・意図  
- 毎回キャラクター説明を書くのは負担  
- ある程度固定した方がAIが安定する  

ユーザー行動  
- 主人公の立場を指定する  
- 名前・性別・年齢の扱いを選択する  

期待される結果  
- セッション開始時の初期条件として使用される  

補足  
- セッション側で上書き可能  

## US-06: シナリオの開始シーンを定義したい

As a シナリオ作者  
I want 開始シーンを固定したい  
So that 毎回同じ導入から物語を始められる

背景・意図  
- 導入は物語体験の印象を決定づける  

ユーザー行動  
- 開始状況を文章で入力する  

期待される結果  
- セッション開始時、最初の Narrative に使用される  

補足  
- 未指定時はAI生成  

## 挿絵設定・プレビュー

## US-11: 挿絵のテイストを指定したい

As a シナリオ作者  
I want 挿絵の画風を指定したい  
So that 文章と視覚表現が一致する

背景・意図  
- 挿絵のズレは没入感を著しく下げる  

ユーザー行動  
- プリセットまたは自由記述で指定  

期待される結果  
- IllustrationConfig に保存される  

補足  
- Narrative設定とは分離  

## US-12: 挿絵の雰囲気（ムード）を指定したい

As a シナリオ作者  
I want 挿絵の感情的トーンを揃えたい  
So that シーンの印象が統一される

ユーザー行動  
- ムードを複数選択  

期待される結果  
- すべての挿絵生成に反映される  

## US-13: 挿絵の禁止要素を指定したい

As a シナリオ作者  
I want 出してほしくない表現を制御したい  
So that 年齢制限・世界観を守れる

ユーザー行動  
- NG要素を入力  

期待される結果  
- ネガティブプロンプトとして使用される  

## US-14: 挿絵を事前にプレビューしたい

As a シナリオ作者  
I want 本番前に挿絵を確認したい  
So that 生成ミスを防げる

ユーザー行動  
- サンプルシーンを入力  
- プレビュー生成を実行  

期待される結果  
- 本番相当の挿絵が表示される  

補足  
- 保存はされない  

## US-15: プレビューを見ながら挿絵設定を調整したい

As a シナリオ作者  
I want 試行錯誤しながら設定を詰めたい  
So that 理想の表現に近づけられる

ユーザー行動  
- 設定変更  
- 再生成  

期待される結果  
- 納得した設定のみが確定される
