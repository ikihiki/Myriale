 # start-session.md
## セッション開始に関するユーザーストーリー

本ドキュメントは、Scenario（シナリオ）から Session（セッション）を開始する一連の体験をユーザーストーリーとして定義する。

前提用語  
- Scenario: ユーザーが事前に作成したシナリオ（設定・雛形）  
- Session: 実際のプレイ単位  
- Narrative: セッション中に生成される語り文章  

---

## US-S01: シナリオから新しいセッションを開始したい

As a プレイヤー  
I want 選択したシナリオから新しいセッションを開始したい  
So that その世界観で物語を体験できる  

背景・意図  
- 同じシナリオを複数回遊びたい  
- プレイの単位としてセッションが必要  

前提条件  
- シナリオが公開または自分用に利用可能である  

ユーザー行動  
1. シナリオ一覧を表示する  
2. 対象シナリオを選択する  
3. 「新しいセッションを開始」を選択する  

期待される結果  
- 新しい Session が作成される  
- Scenario の設定がセッション用にスナップショットされる  
- セッション状態が「Preparing」になる  

補足  
- Scenario の後続変更は既存セッションに影響しない  

---

## US-S02: セッション開始時にシナリオのイントロを見たい

As a プレイヤー  
I want セッション開始時に物語のイントロを読みたい  
So that 世界観や状況を理解してから先に進める  

背景・意図  
- キャラクター確定前に世界観を理解したい  
- イントロは没入感を高める重要な要素である  

前提条件  
- セッションが「Preparing」状態である  

ユーザー行動  
1. セッション開始後、イントロ表示を待つ  
2. イントロ Narrative を読む  

期待される結果  
- 世界観、現在の状況、物語の導入が理解できる  
- 次のステップ（主人公確定）に自然につながる  

非機能・制約  
- イントロは Lore、ジャンル、トーン、開始シーン設定を反映する  
- 主人公は未確定の表現（例：「あなた」「名もなき旅人」）を用いる  
- 初回セッションではスキップ不可  

---

## US-S03: イントロ後に主人公を確定したい

As a プレイヤー  
I want イントロを読んだ後で主人公を確定したい  
So that 世界観に合ったキャラクターで物語を始められる  

背景・意図  
- 事前にキャラを決めると世界観とズレることがある  
- シナリオごとにキャラクターの扱いが異なる  

前提条件  
- イントロ Narrative が表示済みである  
- セッション状態が「Preparing」である  

ユーザー行動（分岐あり）  

パターンA: キャラクター固定  
- シナリオで主人公が定義済み  
- ユーザー操作なしで確定  

パターンB: キャラクター選択式
- 複数の候補キャラクターが必ず提示される
- プレイヤーが1人を選択する
- シナリオ作者が自由生成を許可した場合だけ、候補選択から自由生成へ切り替えられる

パターンC: 自由生成
- 名前、プロフィールなどを入力する
- AIによる入力補助を利用できる
- AI生成は選択肢ではなく自由生成フォーム上の補助ボタンとして提供する
- AI案はプレイヤーが確認・修正してから確定する

シナリオ設定の制約
- 固定式では、候補選択と自由生成を表示しない
- 選択式では、候補選択を必ず利用できる
- 自由生成式では、自由生成以外の方法を表示しない

期待される結果  
- 主人公情報が Session 固有のデータとして確定される  
- 以降の Narrative と選択肢生成に反映される  

非機能・制約  
- AIは自動確定してはならない  
- 確定後の変更は原則不可（将来拡張除く）

## US-S04: セッション開始前に内容を最終確認したい

As a プレイヤー  
I want セッション開始前に設定内容を確認したい  
So that 意図しない条件で物語が始まるのを防げる  

ユーザー行動  
- シナリオ概要、主人公情報、設定内容を確認する  

期待される結果  
- 開始サマリーが表示される  
- 必要に応じて前工程に戻れる  

---

## US-S05: セッションを正式に開始したい

As a プレイヤー  
I want セッションを正式に開始したい  
So that 本編の物語を進められる  

前提条件  
- 主人公が確定している  

ユーザー行動  
- 「物語を始める」を選択する  

期待される結果  
- セッション状態が「Active」になる  
- 本編最初の Narrative が生成される
