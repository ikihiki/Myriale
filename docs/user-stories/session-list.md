# session-list.md
## セッション一覧と継続

## US-SL01: 進行中のセッション一覧を見たい

As a プレイヤー
I want 自分の進行中の物語を一覧で確認したい
So that 続けるセッションを選べる

期待される結果
- 所有する進行中セッションが最終更新日時の新しい順に表示される
- シナリオ名、主人公、ターン、要約、最終更新日時を確認できる

## US-SL02: セッションIDから物語へ戻りたい

As a プレイヤー
I want 一覧のセッションを開きたい
So that 保存された続きから遊べる

期待される結果
- `/sessions/:sessionId` で保存済みターンと進行状態が復元される
- セッションは完了するまでActiveのまま継続できる

## US-SL03: 完了済みの物語も表示したい

As a プレイヤー
I want 必要なときだけ完了済みセッションを一覧に含めたい
So that 過去の物語を探せる

期待される結果
- 既定では進行中セッションだけが表示される
- 「完了済みも表示」を切り替えると本人の完了済みセッションが別セクションに表示される

## US-SL04: 完了済みの物語を読み返したい

As a プレイヤー
I want 完了したセッションのログを開きたい
So that 物語を振り返れる

期待される結果
- 完了済みセッションは同じセッションIDで開ける
- 完了済みセッションでは入力、AI提案、モジュール進行などの変更操作ができない

## US-SL05: 進行中の物語がないとき新しく始めたい

As a プレイヤー
I want シナリオ一覧へ移動したい
So that 新しいセッションを開始できる

期待される結果
- 進行中セッションが0件ならシナリオ一覧への導線が表示される
- 完了済みセッションを表示していても導線は維持される

## US-SL06: 自分のセッションだけを見たい

As a プレイヤー
I want 自分が所有するセッションだけを確認したい
So that 他の利用者の物語が混ざらない

期待される結果
- 一覧と個別表示の両方で所有者が検証される
- 未認証の場合はセッション情報を取得できない
