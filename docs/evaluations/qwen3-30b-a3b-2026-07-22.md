# qwen3-30b-a3b real-provider evaluation — 2026-07-22

## Scope

- Provider: Runpod OpenAI-compatible endpoint
- Model: `qwen3-30b-a3b`
- Prompt version: `narrative-prompt.v3`
- Context version: `narrative-context.v2`
- Dialogue schema: `narrative-dialogue.v8`
- Repetitions: 3 per case
- Evaluated cases: Player agency, clarification, NPC lore/voice, Session state, Module authority, forbidden paraphrase, detailed NPC reply, 20+ Turn NPC continuity, and positive/negative progression signals
- Secret handling: endpoint, API key, prompts, Player text, raw narratives, raw provider responses, and request IDs were not recorded

## Production safeguards established during evaluation

Early runs showed that the model was not reliable enough to own the six-field application envelope. The production path was therefore changed without lowering the approval thresholds:

- Qwen3 thinking is disabled so hidden reasoning does not consume the output budget.
- The provider returns only the Narrative body; schema version, turn type, heading, interpretation, and progression signals are server-owned.
- Progression signals require deterministic server evidence and Forbidden Facts use normalized semantic guards.
- A whole-request token budget retains authoritative context while reducing lower-priority history.
- Schema-invalid or semantically unsafe provider bodies use a bounded, context-grounded safe fallback. Provider timeout, 429, 5xx, credential, and content-policy failures remain visible and never become a successful Turn.
- An unmeasured warmup phase handles the expected several-minute cold start before fixed quality measurements begin.

## Final production-gate result

The final sanitized run completed 30 measured generations after a successful 2,197 ms warmup:

- Schema success: **30/30 (100%)**
- Quality success: **30/30 (100%)**
- Signal accuracy: **30/30 (100%)**
- Every case: **3/3 schema and quality success**
- Average measured latency: **44,188 ms**
- Input tokens: **182,597**
- Output tokens: **34,357**

A separate secret-enabled integration run also completed two consecutive real-provider turns through Session API acceptance, hosted worker execution, persistence, reload, context-hash change, and Turn chaining.

## Disposition

**Approved for production Narrative generation with the server-owned controls, semantic guards, request budget, grounded fallback, and cold-start warmup described above.** Approval applies to this complete application configuration, not to unconstrained direct use of the model.

The fixed production gate remains at 95% schema success, 90% quality success, 95% signal accuracy, at least 2/3 success per case, and at least three repetitions per case. These thresholds were not lowered to obtain approval.
